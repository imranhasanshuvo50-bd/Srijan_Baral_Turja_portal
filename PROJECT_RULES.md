# Project Rules

Codex must read and follow this file first before making any code or design changes in this project.

## Engineering Standards

- All components must be reusable.
- Follow Clean Architecture principles.
- Use mobile-first responsive design.
- Maintain accessibility, including ARIA labels, semantic HTML, focus states, and keyboard navigation.
- Optimize for performance.
- Add and maintain SEO metadata where relevant.
- Keep comments minimal, but meaningful when they clarify non-obvious code.
- Organize Tailwind utility classes consistently and readably.
- Avoid hardcoded content in components; load content from the `src/data` layer instead.
